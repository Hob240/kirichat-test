Penjelasan singkat (bahasa awam):
- src/app: Tempat semua halaman website. Folder (dashboard) berisi halaman setelah login (ada sidebar). Folder (auth) untuk login/register.
- src/components: Kumpulan “kepingan UI” yang bisa dipakai ulang (tombol, kartu, tabel, modal, dll.).
- src/store: Penyimpanan data sementara di front-end (biar halaman terasa hidup walau belum ada backend).
- src/lib dan src/services: Otak kecil & jembatan ke layanan luar (Supabase/AI/Activepieces). Kita siapkan dulu kerangkanya.
- docs: Dokumen panduan internal tim. Biar semua satu visi.

Frontend vs Backend (bahasa awam):
- Frontend (tampilan/yang dilihat dan diklik user):
  - public/
  - src/app/(auth)/**/page.tsx
  - src/app/(marketing)/page.tsx
  - src/app/(dashboard)/**/page.tsx dan layout.tsx
  - src/components/**
  - src/hooks/**
  - src/store/**
  - src/mocks/**
  - src/i18n/**
  - app/globals.css (atau styles/ jika dipakai)

- Backend (mesin/pengelola data di server):
  - src/app/api/**/route.ts (endpoint API berjalan di server)
  - src/services/** (supabase.ts, llm/, billing/polar.ts, actions/activepieces.ts)
  - middleware.ts (penjaga pintu global; bisa untuk redirect auth/i18n)
  - scripts/** (alat bantu yang jalan di server/terminal)
  - .env.local (menyimpan API Key dan rahasia)

- Bersama/Umum (dipakai Frontend dan/atau Backend):
  - src/lib/** (utils, constants, validators, security rules UI)
  - src/types/** (bentuk data/tipe agar konsisten)
  - docs/** (dokumentasi internal)


Stack yang dipakai (ringkas):
- Frontend:
  - Next.js 14 (App Router) + TypeScript
  - Tailwind CSS (styling)
  - Zustand (state management ringan untuk MVP)
  - Komponen UI kustom (gaya shadcn/ui kompatibel)
- Backend:
  - Supabase: PostgreSQL (database), Auth, Storage, RLS (Row-Level Security), pgvector (vector DB untuk RAG)
  - LLM Providers via adapter: OpenAI (GPT-4o/5/mini), Google Gemini, DeepSeek
  - Activepieces (self-hosted) untuk automation/actions
  - Billing: Polar.sh (menghubungkan Stripe/Apple Pay/Google Pay/PayPal)
- Keamanan (desain):
  - Enkripsi in-transit (TLS) dan at-rest (DB/Storage)
  - Isolasi data per workspace via RLS
  - Webhook HMAC + anti replay, rate limiting, dan CSP ketat
- Dev Tooling:
  - ESLint, Prettier, PostCSS, Tailwind CLI
  - .env.local untuk secrets (jangan di-commit)
